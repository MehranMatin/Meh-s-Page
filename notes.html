Notes

  • the browser's event object will have a target property that tells us exactly which HTML element was interacted with
  • If you forget which properties each api object has, you can check the Preview panel in the DevTools Network tab, load the requested URL in another browser tab, or console.log() the response data.
  • When using fetch() to create a request, the request might go one of two ways:
    The request may find its destination URL and attempt to get the data in question, which would get returned into the .then() method;
    if the request fails, that error will be sent to the .catch() method.
  • Check if request was successful by using the ok property that's bundled in the response object from fetch(). When the HTTP request status code is something in the 200s, the ok property will be true.
  • event.preventDefault() stops the browser from performing the default action the event wants it to do. In the case of submitting a form, it prevents the browser from sending the form's input data to a URL, as we'll handle what happens with the form input data ourselves in JavaScript.
  • Asynchronous communication with a server is often referred to as AJAX, which stands for Asynchronous JavaScript and XML. The XML in this term refers to an old-fashioned way of formatting data. XML has been largely replaced by JSON, but the name has persisted.
  • When writing href filepaths, a forward slash "/" will always represent the path to the topmost directory of an application or project. When user clicks the <a> element, they will be taken to the topmost directory.
    Because there is no file specified, the index.html file will be loaded.
  • HTML defines a webpage's structure, CSS defines the design, and JavaScript defines the behavior.
  • Separation of concerns,: It's better to keep code that serves different purposes in separate files so that it's easier to maintain.
  • hero image refers to a large banner image placed prominently at the top of the page. The placement has a specific task: to pique the visitor's curiosity enough to get them reading, scrolling, clicking, and engaging with the website. Think of the hero section as the banner, headline, or billboard for a website.
  • Creating the skeleton of an HTML document is also known as "scaffolding".
  • Because the <title> element's content appears in the browser's tab, it's good practice to structure the content as [page title] - [site title]. Page titles should be descriptive but also concise because Google cuts off search result titles at around 60 characters.
  • A set of HTML code that builds out a part of the overall page's UI is commonly referred to as a "component". The idea behind a component is that it can be reused in multiple places throughout a site.
  • Right-click the page and select "View page source" to see its HTML.

Saved Terminal Codes

Checks current directory for any saved .DS_Store files and removes them from the git repository

find . -name .DS_Store -print0 | xargs -0 git rm -f --ignore-unmatch

Checks current git configurations

git config --list

Brings remote references up to date.

git remote update

Tells whether the branch you are tracking is ahead, behind or has diverged. If it says nothing, the local and remote are the same.

git status -uno

Shows the commits in all of the branches whos names end in 'main'.

git show-branch *main

Tells which remote links, if any, you've established for the project.

git remote -v

Use nano in command line to edit text files

nano file.txt

To change Git remote URL

$ git remote set-url origin https://git-repo/new-repository.git

How to get all remote branches into local repo

for remote in `git branch -r`; do git branch --track ${remote#origin/} $remote; done

How to add space tile to mbp dock

defaults write com.apple.dock persistent-apps -array-add '{"tile-type"="spacer-tile";}'; killall Dock
