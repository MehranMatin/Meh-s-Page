Notes

  • The value of the href used here—a forward slash ("/")—will always represent the path to the topmost directory of an application or project. In this case, when a user clicks the <a> element, they will be taken to the topmost directory. And because there is no file specified, the index.html file will be loaded.

  • HTML defines a webpage's structure, CSS defines the design, and JavaScript defines the behavior.

  • Separation of concerns,: It's better to keep code that serves different purposes in separate files so that it's easier to maintain.

  • hero image refers to a large banner image placed prominently at the top of the page. The placement has a specific task: to pique the visitor's curiosity enough to get them reading, scrolling, clicking, and engaging with the website. Think of the hero section as the banner, headline, or billboard for a website.

  • Creating the skeleton of an HTML document is also known as "scaffolding".

  • Because the <title> element's content appears in the browser's tab, it's good practice to structure the content as [page title] - [site title]. Page titles should be descriptive but also concise because Google cuts off search result titles at around 60 characters.

  • A set of HTML code that builds out a part of the overall page's UI is commonly referred to as a "component". The idea behind a component is that it can be reused in multiple places throughout a site.

  • Right-click the page and select "View page source" to see its HTML.

Saved Terminal Codes

Checks current directory for any saved .DS_Store files and removes them from the git repository

find . -name .DS_Store -print0 | xargs -0 git rm -f --ignore-unmatch

Checks current git configurations

git config --list

Brings remote references up to date.

git remote update

Tells whether the branch you are tracking is ahead, behind or has diverged. If it says nothing, the local and remote arethe same.

git status -uno

Shows the commits in all of the branches whos names end in 'main'.

git show-branch *main

Tells which remote links, if any, you've established for the project.

git remote -v

Use nano in command line to edit text files

nano file.txt
